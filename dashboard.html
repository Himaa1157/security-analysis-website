üîç Identifikasi Kerentanan

KRITIS

1. Autentikasi Client-Side

Deskripsi: Proses validasi login dilakukan sepenuhnya di sisi client menggunakan JavaScript

Dampak: Penyerang dapat memodifikasi kode JavaScript atau melewati validasi dengan mudah

Lokasi: File index.html, fungsi login()

üõ°Ô∏è Rekomendasi Perbaikan:
  • Pindahkan validasi ke server-side
  • Gunakan framework backend yang aman
  • Implementasi session management yang proper
KRITIS

2. Penyimpanan Password Plain Text

Deskripsi: Password disimpan dan dibandingkan dalam bentuk plain text

Dampak: Jika database bocor, semua password dapat langsung dibaca

Lokasi: File index.html, variabel validUsers

üõ°Ô∏è Rekomendasi Perbaikan:
  • Gunakan hashing algorithm (bcrypt, Argon2)
  • Tambahkan salt untuk setiap password
  • Implementasi pepper key
// Contoh implementasi bcrypt
const bcrypt = require('bcrypt');
const saltRounds = 12;

// Hash password
const hashedPassword = await bcrypt.hash(password, saltRounds);

// Verify password
const match = await bcrypt.compare(inputPassword, hashedPassword);
TINGGI

3. Session Management yang Tidak Aman

Deskripsi: Status login disimpan di localStorage yang rentan terhadap XSS

Dampak: Penyerang dapat mencuri session token melalui XSS attacks

Lokasi: localStorage.setItem('loggedIn', 'true')

üõ°Ô∏è Rekomendasi Perbaikan:
  • Gunakan httpOnly cookies untuk session
  • Implementasi CSRF protection
  • Set proper session timeout
SEDANG

4. Tidak Ada Rate Limiting

Deskripsi: Tidak ada pembatasan percobaan login yang gagal

Dampak: Memungkinkan brute force attacks

üõ°Ô∏è Rekomendasi Perbaikan:
  • Implementasi rate limiting (max 5 percobaan per IP)
  • Tambahkan CAPTCHA setelah beberapa percobaan gagal
  • Lock account sementara setelah terlalu banyak percobaan
SEDANG

5. Tidak Menggunakan HTTPS

Deskripsi: Komunikasi antara client dan server tidak dienkripsi

Dampak: Rentan terhadap man-in-the-middle attacks

üõ°Ô∏è Rekomendasi Perbaikan:
  • Gunakan SSL/TLS certificate
  • Implementasi HSTS header
  • Redirect semua traffic ke HTTPS

üéØ Rekomendasi Implementasi Keamanan

Arsitektur Keamanan yang Direkomendasikan

  1. Backend Server: Gunakan Node.js/Express dengan middleware security
  2. Autentikasi: Implementasi JWT dengan httpOnly cookies
  3. Database: Gunakan PostgreSQL/MySQL dengan prepared statements
  4. Password Hashing: bcrypt dengan cost factor 12+
  5. Input Validation: Library seperti Joi atau Validator.js
// Contoh middleware keamanan di Express.js
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');

app.use(helmet());
app.use(rateLimit({
windowMs: 15 * 60 * 1000, // 15 menit
max: 100 // maksimal 100 request per windowMs
}));

üìù Kesimpulan Analisis

Website ini sengaja dibuat dengan berbagai kerentanan keamanan untuk tujuan edukasi. Dalam environment produksi, implementasi seperti ini akan sangat berbahaya karena:

  • Membuka peluang data breach dan pencurian kredensial
  • Rentan terhadap session hijacking
  • Dapat menyebabkan unauthorized access ke sistem
  • Melanggar compliance regulations (GDPR, PDPA, dll.)

Peringatan: Jangan pernah menggunakan kode seperti ini dalam sistem produksi. Selalu ikuti best practices keamanan dan lakukan security testing secara berkala.
